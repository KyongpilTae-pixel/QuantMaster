### 2. 전체 async 핸들러 State 락 블로킹 완전 해소

**문제**: 스캔 중(밸류돌파 등) 탭 전환 시 응답이 느리거나 앱이 잠시 멈추는 현상

**근본 원인**: 모든 `async def` 이벤트 핸들러는 실행 중 State 락을 점유함.
기존 polling 타임아웃이 3.0초로 설정되어 탭 전환 이벤트가 최대 3초 지연됨.
또한 일부 핸들러에서 `asyncio.to_thread()`를 직접 await하여 락을 수십 초 점유함.

**수정 1 — polling 타임아웃 3.0s → 0.5s (전체)**
```python
# 변경 전
await asyncio.wait_for(asyncio.shield(_task), timeout=3.0)
# 변경 후
await asyncio.wait_for(asyncio.shield(_task), timeout=0.5)
```
적용 범위: 전체 10개 polling loop (do_fetch_leaders 1D, run_scan 5개, run_backtest 등)

**수정 2 — 세력탐지 직접 await 해소 [CRITICAL]**
```python
# 변경 전: State 락 점유 30초+
ctx = await asyncio.to_thread(scanner.prepare, ...)
new = await asyncio.to_thread(scanner._scan_batch, ...)

# 변경 후: create_task + 0.5s polling
_prep_task = asyncio.create_task(asyncio.to_thread(scanner.prepare, ...))
while not _prep_task.done():
    try:
        ctx = await asyncio.wait_for(asyncio.shield(_prep_task), timeout=0.5)
        break
    except asyncio.TimeoutError:
        yield
```

**수정 3 — CRITICAL: bare fdr.DataReader() 동기 호출 해소**
```python
# 변경 전: Python 이벤트루프 전체 차단 (WebSocket 포함)
idx_df = fdr.DataReader(code, start, end)

# 변경 후: asyncio.to_thread + polling
_t = asyncio.create_task(asyncio.to_thread(fdr.DataReader, code, start, end))
while not _t.done():
    ...
```

## 기술 참고

- Reflex 0.8.28 State 락: `async def` 이벤트 핸들러는 실행 중 항상 락 점유
  - `yield` 시 순간적으로 해제 → 대기 중인 이벤트(탭 전환 등) 처리
  - `@rx.event(background=True)` 만 락 없이 실행
- `asyncio.to_thread()` 직접 await는 thread 완료까지 락 유지 → 탭 전환 차단
- 해결 패턴: `create_task + wait_for(shield, 0.5) + yield` → 0.5초마다 락 해제
- `fdr.DataReader()` bare 호출은 Python 이벤트루프 전체를 차단 → 반드시 to_thread 감싸기
