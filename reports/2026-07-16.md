### 3. 성과추적 탭 UI 블로킹 근본 수정

**원인**: `load_tracker_picks`가 `async def`이지만 `yield`가 없어 Reflex 상태 락을 보유한 채 `asyncio.to_thread()`를 await함. DB 조회 시간 동안 다른 이벤트 처리 불가 → UI 전체 응답 없음.

**수정**: `yield` 1줄 추가 → async generator 전환 → DB 조회 전 상태 락 해제.

```python
async def load_tracker_picks(self):
    yield  # 상태 락 해제 → 탭 전환 등 다른 이벤트 처리 가능
    picks = await asyncio.to_thread(load_tracked_picks, 30, mode, market)
    ...
```

### 5. 모든 스캔 모드 `@rx.event(background=True)` 전환

**문제**: 역발상 과매도 스캔 실행 시 다른 탭이 동작하지 않음.

**근본 원인**: `async def run_scan(self)` 방식의 while loop 패턴은 `asyncio.wait_for(..., timeout=0.5)` 동안 Reflex state lock을 500ms씩 점유. 이 창 동안 다른 이벤트(탭 전환 등) 처리 불가. 특히 `dividend` 스캔은 while loop조차 없어 단일 `await asyncio.to_thread(...)` 전체 실행 시간 동안 state lock 보유.

**해결**: 8개 스캔 모드를 모두 `@rx.event(background=True)` background task로 전환.

| 스캔 모드 | 변경 전 | 변경 후 |
|-----------|---------|---------|
| pullback | while loop (500ms lock 창) | background task |
| trend | while loop (500ms lock 창) + 진행상황 | background task + asyncio.sleep 폴링 |
| defensive | while loop (500ms lock 창) | background task |
| stock_momentum | while loop (500ms lock 창) | background task |
| mean_reversion | while loop (500ms lock 창) | background task (이미 이전 세션 반영) |
| magic_formula | while loop (500ms lock 창) | background task |
| **dividend** | **단일 await (전체 실행 시간 lock)** | **background task (CRITICAL)** |
| quant | while loop (500ms lock 창) | background task |
| whale | 유지 (복잡한 중단 메커니즘) | 미변경 |

**background task 패턴**:
- `async with self:` 블록에서만 순간적으로 state lock 획득
- `await asyncio.to_thread(scan_fn, ...)` 동안 state lock 완전 해제
- 다른 탭 전환, 작업현황 확인 등 모든 이벤트 자유롭게 처리
- trend 스캔: `asyncio.create_task + asyncio.sleep(0.5) 폴링`으로 진행 상황 표시 유지

**작업현황 탭 동작**: `is_scanning=True` 동안 "역발상 과매도 스캔 실행 중" 카드 표시 — background task 전환 후 탭 전환 자유로워짐.
