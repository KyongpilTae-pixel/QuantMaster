### 2. 다중 팩터 모델 Phase 3 — Magic Formula 스캐너 (Greenblatt)

**`utils/magic_formula_scanner.py`** (신규):

전략: EBIT/EV(수익률) 순위 + ROIC(자본효율) 순위 → 합산 순위 낮을수록 고수익+저평가 종목.

```
EV      = 시가총액 + 총부채 - 현금성자산
ROIC    = EBIT / (NWC + 순고정자산)
          NWC = max(0, 유동자산 - 유동부채)
          순고정자산 = 유형자산 순액(Net PPE)
```

구현 세부:
- `_calc_magic(args)`: 단일 종목 지표 계산, EBIT≤0/EV≤0/IC≤0이면 None 반환
- `scan_magic_formula(market, min_mktcap_eok, top_n=30, max_universe=150, _timeout_s=120)`:
  - KR: `fetch_kr_stock_listing` → 시총 필터 → 상위 150개
  - US: `fdr.StockListing("S&P500")` → 상위 150개
  - `ThreadPoolExecutor(max_workers=8)` 병렬 수집
  - EY 내림차순 순위 → ROIC 내림차순 순위 → 합산 오름차순 정렬
  - bool 플래그: `ey_top`(상위 25%), `roic_top`(상위 25%)
  - 타임아웃 초과 종목 → `warning` 메시지에 카운트 표시

**`main/main.py`** 추가:
- State: `magic_results: List[dict]`, `magic_min_mktcap: int = 3_000`
- `set_magic_min_mktcap()` setter
- `run_scan()` 분기: `scan_mode == "magic_formula"` → 0.5s polling, 120s 타임아웃
- `magic_opts` hstack: 시총 3천억+/1조+/전체 버튼
- 사이드바 옵션 cond 체인: `mean_reversion → mr_opts → magic_formula → magic_opts`
- Callout (teal 테마): Greenblatt Magic Formula 전략 설명
- `rx.select.item("매직 포뮬러", value="magic_formula")` 드롭다운 추가
- `magic_formula_table()` UI 함수: 순위/종목명/합산순위/EY순위/ROIC순위/EY%/ROIC%/현재가/시가총액/[분석]
- `scanner_tab()` cond 체인에 magic_formula 레벨 추가

**scanner_tab() cond 구조 (완성)**:

## 기술 참고

- **yfinance 필드명 불일치 대응**: `_get(df, [key1, key2, ...], col)` 헬퍼로 여러 키명 순차 시도
- **CapEx 부호 관행**: yfinance는 CapEx를 음수로 보고하는 경우가 대부분이나 양수 케이스도 있음 → `fcf > ocf`이면 부호 반전
- **scanner_tab() 닫는 괄호 관리**: cond 레벨 추가 시 매 레벨마다 닫는 괄호 1개 추가 필요 (7레벨 → 8레벨 시 `)` 1개 추가)
- **Reflex `dict` State**: `value_metrics: dict = {}` — `rx.foreach` 없이 `State.value_metrics["key"]` 직접 참조 가능
- **Magic Formula 적격 조건**: EBIT > 0, EV > 0, ROIC > 0 (투하자본 > 0) — 3개 모두 미충족 시 해당 종목 제외
