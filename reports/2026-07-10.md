## 작업 요약

### 1. WebSocket 끊김 + 탭 전환 시 앱 무응답 문제 근본 해결

**증상**: 퀀트 스캔/주도주 로드 중 탭 전환하면 UI 동결 → 새로고침해도 복구 안 됨

**근본 원인**: Reflex에서 `async def` 이벤트 핸들러는 실행 중 State 락을 점유함.
`asyncio.to_thread(...)` 중에도 락을 유지하므로 수십 초~수 분간 WebSocket 이벤트가 큐에 쌓임
→ 브라우저 WebSocket 타임아웃(os error 10053) → 재연결 시 State 락에 막혀 실패

---

#### 1-1. `do_prefetch_momentum_bg` — `@rx.event(background=True)` 적용

KOSPI/KOSDAQ 90초 OHLCV 수집이 State 락을 점유하던 문제.
Reflex 0.8.28의 `@rx.event(background=True)` 데코레이터로 완전 해결:
- 함수 전체가 락 없이 실행
- 상태 변경은 `async with self:` 블록에서만 순간적으로 락 획득

```python
@rx.event(background=True)
async def do_prefetch_momentum_bg(self):
    async with self:
        self.leaders_prefetch_status = "캐싱 중..."
    data = await asyncio.to_thread(scan_stock_momentum_all_periods, ...)
    async with self:
        self._refresh_momentum_cache_status()
```

**주의**: `rx.background`는 0.8.28에 없음 → `rx.event(background=True)` 사용

#### 1-3. `run_scan` (quant/pullback/defensive/stock_momentum 브랜치)

1~5분짜리 단일 `asyncio.to_thread()` 호출 → `create_task + polling` 패턴으로 전환:
3초마다 `yield`로 락을 풀어 WebSocket 유지.
추세추종(trend) 브랜치는 이미 동일 패턴 적용돼 있어 변경 없음.

```python
_task = asyncio.create_task(asyncio.to_thread(scan_func, ...))
while not _task.done():
    try:
        raw = await asyncio.wait_for(asyncio.shield(_task), timeout=3.0)
        break
    except asyncio.TimeoutError:
        yield  # 3초마다 락 해제 → WebSocket 유지
else:
    raw = _task.result()
```

## 기술 참고

- Reflex 0.8.28: `@rx.event(background=True)` — State 락 없이 실행, `async with self:` 로 상태 갱신
- `asyncio.create_task + wait_for(shield(task), timeout=3.0) + yield` — 기존 async generator 핸들러에서 WebSocket 유지하는 polling 패턴
- WebSocket os error 10053 = 호스트 소프트웨어에 의한 연결 중단 (State 락 점유 중 타임아웃)
