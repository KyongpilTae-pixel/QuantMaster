01
다중 팩터 모델 (Multi-Factor)
현재: PBR + GPA(Gross Profitability) 2개 팩터만 사용
Piotroski F-Score
우선순위 상구현 쉬움
재무 건전성 9개 항목 스코어링. 현재 PBR/GPA만으로 놓치는 "싼데 부실한" 종목 제거에 탁월.
  • ROA 증가 여부 (순이익/총자산)
  • 영업현금흐름 양수 여부
  • 부채비율 감소 여부
  • 유동비율 개선 여부
  • 신주 발행 없음 여부
  • GPM 개선 / 자산회전율 개선
격차: yfinance financials로 계산 가능. KR은 NAVER 재무제표 추가 수집 필요
Magic Formula (Greenblatt)
우선순위 상구현 쉬움
EBIT/EV (이익수익률) × ROIC (투자자본수익률) 순위 합산. 가치+수익성을 동시에 포착.
  • EBIT / Enterprise Value 순위
  • EBIT / (순운전자본 + 고정자산) 순위
  • 두 순위 합산 → 상위 30-50개 유니버스 선정
  • 연 1회 리밸런싱
격차: EV 계산 (시총 + 순부채) 로직 추가 필요. PBR 필터 대체 또는 병행 가능
EV/EBITDA · P/FCF
우선순위 중구현 쉬움
PBR/PER보다 자본 구조 왜곡에 덜 민감한 밸류에이션 지표. 특히 부채 많은 섹터에서 유용.
  • EV/EBITDA < 섹터 중앙값 필터
  • P/FCF: 자유현금흐름 기반 밸류
  • PBR이 높은 미국주 분석에 효과적
격차: yfinance info에 enterprise_value, ebitda, free_cashflow 있음. 즉시 추가 가능
Quality Factor 복합
우선순위 중구현 보통
ROE 안정성(5년 표준편차), 이익 일관성, 낮은 레버리지를 결합한 퀄리티 스코어.
  • ROE 변동성 (낮을수록 고품질)
  • 5년 연속 순이익 양수 여부
  • 부채/자본 비율 < 1.0
  • ROIC - WACC 스프레드 (경제적 해자)
격차: 분기별 재무 데이터 5년치 수집 및 저장 인프라 필요
02
포트폴리오 최적화 (Portfolio Optimization)
현재: 고정 3분할 매수 플랜 (30% / 30% / 40%) 만 존재
Risk Parity (위험 균형 배분)
우선순위 상구현 보통
각 자산이 포트폴리오 전체 변동성에 동일하게 기여하도록 비중 역산. 변동성 큰 자산은 자동으로 비중 줄어듦.
  • 개별 변동성 역수로 비중 계산 (1/σ)
  • 상관관계 고려 시 → 마코위츠 공분산 행렬
  • 글로벌 모멘텀 탭 자산에 즉시 적용 가능
격차: numpy covariance 계산으로 구현 가능. 기존 VAA 전략과 비교 백테스트 권장
Kelly Criterion 포지션 사이징
우선순위 상구현 쉬움
백테스트 승률과 손익비로 최적 배팅 비율 계산. 반켈리(50%)로 실전 적용.
  • f* = (bp - q) / b (b=손익비, p=승률, q=패율)
  • 분할 매수 플랜에 반영 가능
  • 고MFI 종목 → 반켈리 축소 자동 적용
격차: 백테스트 결과(승률·손익비)가 이미 있으므로 수식 추가만으로 즉시 구현 가능
Markowitz 효율적 프론티어
우선순위 중구현 어려움
최대 샤프비율 포트폴리오, 최소 분산 포트폴리오를 scipy 최적화로 계산.
  • 보유 종목 상관관계 히트맵
  • 효율적 프론티어 곡선 시각화
  • 섹터 집중도 제약 조건 추가 가능
격차: scipy.optimize.minimize + 보유종목 OHLCV 병렬 수집. 보유종목분석 탭 확장
드로우다운 서킷브레이커
우선순위 상구현 쉬움
포트폴리오 전체 평가손실이 임계값 도달 시 신규 진입 자동 차단 및 경고.
  • 포트폴리오 손익률 –10% → 황색경보
  • –15% → 신규 스캔 비활성화 권고
  • –20% → 풀 현금 전환 알림
격차: 보유종목분석 손익률(이미 있음)에 임계값 체크 로직만 추가
03
변동성 기반 전략 (Volatility-Based)
현재: Beta 계산, 연환산 변동성(60일×√252) — 리스크 관리에 직접 미사용
변동성 타겟팅 (Vol Targeting)
우선순위 상구현 보통
목표 변동성(예: 연 15%)을 유지하도록 포지션 크기를 자동 조절. 고변동성 구간에서 자동으로 노출 축소.
  • 포지션 비율 = 목표변동성 / 실현변동성(20일)
  • VKOSPI > 30 시 추가 축소 (패닉 구간)
  • 분할 매수 플랜 비율에 반영
격차: 변동성 계산 이미 있음. 타겟 비율 설정 UI + 전략 엔진 연동 필요
ATR 기반 손절 / 포지션 사이징
우선순위 상구현 쉬움
현재 고정 VWAP×0.96 손절을 ATR(14일 평균진폭) 배수로 대체. 종목별 변동성에 맞춘 손절.
  • ATR = 14일 True Range 평균
  • 손절선 = 진입가 – 2.5 × ATR14
  • 포지션 크기 = 위험금액 / (2.5 × ATR14)
격차: indicators.py에 ATR 함수 추가 → 전략 엔진 + 분석 탭 연동
VKOSPI 공포지수 모니터링
우선순위 상구현 쉬움
한국 시장의 옵션 내재변동성 지수(VKOSPI). 30 초과 = 패닉 구간, 역발상 매수 시그널.
  • FDR: "VKOSPI" 코드로 데이터 수집
  • 장세 판단 섹션에 통합
  • VKOSPI > 30 시 "과매도 반등 스캔" 권고
격차: 데이터 수집 1줄. market_regime.py 또는 주간리포트에 즉시 추가 가능
Donchian Channel 돌파 전략
우선순위 중구현 쉬움
20일 고가/저가 채널. 채널 상단 돌파 = 추세 진입. CTA(추세추종) 펀드의 가장 기본 전략.
  • 진입: 종가 > max(High, 20일)
  • 청산: 종가 < min(Low, 10일)
  • 추세추종 스캔과 통합 가능
격차: indicators.py 함수 추가 + 추세추종 브랜치 필터로 선택 가능
04
통계적 차익거래 (Statistical Arbitrage)
현재: 없음 — 단방향 롱 전략만 존재
Pairs Trading (페어 트레이딩)
우선순위 중구현 어려움
같은 섹터 내 공적분(cointegration) 관계 종목 쌍 발굴. 스프레드 Z-score 임계값에서 롱/숏 동시 진입.
  • Engle-Granger 공적분 검정
  • 스프레드 Z-score > 2.0 → 숏/롱
  • Z-score → 0 수렴 시 청산
  • 유사 섹터 ETF로 개인투자자 적용 가능
격차: statsmodels 라이브러리 필요. 쌍 탐색에 O(n²) 계산 → 섹터 내 제한 탐색으로 최적화
RSI 평균회귀 전략
우선순위 중구현 쉬움
RSI < 25 & 볼린저밴드 하단 이탈 → 평균회귀 매수. 현재 추세추종과 방향 반대의 단기 전략.
  • RSI14 < 25 + 종가 < BB 하단(2σ)
  • 목표: RSI 50 회복 or BB 중간밴드
  • 고변동성 구간에서 수익성 높음
격차: RSI 지표 추가(미구현) + BB 지표. 스캔 모드에 "역발상(Mean-Reversion)" 탭 신설 가능

이벤트 드리븐 (Event-Driven)
현재: 가격/재무 데이터만. 이벤트 캘린더·공시 연동 없음
PEAD (어닝 드리프트 추종)
우선순위 중구현 보통
어닝 서프라이즈(실적 예상 초과) 발표 후 주가가 같은 방향으로 계속 드리프트하는 현상.
  • 실적 발표일 + 예상치 대비 실제 EPS 비교
  • 서프라이즈 > 5% → 다음날 진입
  • 3-4주 보유 후 청산
  • US: yfinance earnings_history 활용 가능
격차: KR은 FnGuide/에프앤가이드 공시 API 또는 DART API 필요. US는 yfinance로 부분 구현 가능
52주 신고가 브레이크아웃
우선순위 상구현 쉬움
52주 최고가 돌파 종목은 강한 모멘텀 지속성을 보임. 현재 추세추종 스캔과 연계 가능.
  • 종가 > max(Close, 252일) 조건 추가
  • 돌파 당일 거래량 ≥ 20일 평균 2배
  • 기존 스캔 결과에 "신고가 뱃지" 표시
격차: OHLCV 이미 있으므로 252일 rolling max 계산만 추가. 즉시 구현 가능

수급 분석 & 크로스 자산 상관관계

외국인·기관 순매수 추적

USD/KRW → KOSPI 방향성 연계

원/달러 환율과 KOSPI의 역상관 관계 활용. 환율 급등 = 외국인 이탈 신호로 매크로 필터링.

격차: report_generator.py에 이미 USD/KRW 수집 코드 있음. 판단 로직만 추가
공매도 잔고 변화율 추적

머신러닝 예측 모델
현재: 없음 — 규칙 기반 필터만 사용
XGBoost 상승 확률 분류기
우선순위 하구현 어려움
PBR·MFI·OBV·모멘텀 등 기존 지표를 피처로 → 30일 후 +5% 이상 상승 여부 분류.
  • 피처: PBR, GPA, MFI, OBV기울기, 모멘텀1M/3M, Beta
  • 레이블: 30일 후 수익률 > 5%
  • Walk-forward 교차검증 필수
  • 스캔 결과에 "ML 확률 스코어" 컬럼 추가
격차: xgboost/lightgbm 라이브러리. 학습 데이터셋 구축 (과거 스캔 기록 활용)
피처 중요도 시각화
우선순위 하구현 보통
ML 모델의 feature importance로 "현재 시장에서 어떤 지표가 가장 예측력 높은지" 시각화.
  • SHAP value 기반 설명 가능 AI
  • 시장 레짐별 지표 중요도 변화 추적
  • 월간 리포트에 "이달의 핵심 팩터" 섹션
격차: shap 라이브러리. ML 모델 완성 후 단계

고급 백테스팅
현재: 단일 종목 VWAP 전략, 단순 수익률 계산. 거래세·슬리피지 미반영
Walk-Forward Optimization
우선순위 중구현 보통

  • Training 6개월 → Testing 1개월 → 롤링
  • 각 구간 최적 파라미터 자동 탐색
  • 단일 최적화 대비 실제 성과 예측력 높음
격차: backtester.py 재설계 필요. 기간 분할 + 파라미터 그리드 서치 구조
